virtual ARGS_TYPE* SQLActionFactory< CODE_TYPE, ARGS_TYPE, OUT_TYPE >::getArgs ( CODE_TYPE *  code)
pure virtualinherited

This method have to be implemented using the specific ODBC API in use. This should return a result (typically a string) from an error code (typically an int).

Parameters
theerror code
Returns
the message object returned by the DB corresponding to the error code (passed in 'err').

The resulting MSG_TYPE will be passed to the ActionList as argument so if you want to implement actions by your self, be sure to set: SQLAction<IN == MSG_TYPE, [OUT is optional]> Note also that for each MSG_TYPE returned an entire ActionList is executed using the same MSG_TYPE message.

You could also use your own MSG_TYPE type passing to Actions an object to work on. In this case you can implement your own Actions starting from SQLAction. NOTE: this is not a need since SQLxxxAction accept MSG_TYPE.

NOTE: if your ODBC API don't use this method you could use an ERROR_TYPE equals to the MSG_TYPE implementing this method simply returning the input 'err' variable.

Pointer to the method used to get codes. This is probably a function coming from your ODBC API implementation of the ERROR_TYPE SQLConnector::getError()

NOTE: Never return a NULL pointer. NOTE: ERROR_TYPE shouldn't contain *

We can change this behavior but we will lost the possibility to run the 'last' application_list associated to the stop error code.

virtual bool SQLActionFactory< CODE_TYPE, ARGS_TYPE, OUT_TYPE >::stop ( CODE_TYPE *  code)
pure virtualinherited

This method is used by the SQLExceptionManager to stop requesting for next error to the DB server.

Note that you can still apply an action_list to this error code. Actions execution will stop at the and of that list.

Parameters
theerror code corresponding to the normal status (or equivalent).
Returns
true if next call of getCode shouldn't be done, stopping actions.
