Detailed Description

template<class ERROR_TYPE, class MSG_TYPE, class OUT = void>
class SQLErrorFactory< ERROR_TYPE, MSG_TYPE, OUT >

Implementation of an ActionFactory used to easily build an {Static|Object|Dynamic}ErrorFactory, depending from the generated Actions type (by getActions()).


To build this class you have to:

  • implement some methods defined by an ActionFactory:
  • getActions
  • stop
  • implement an SQLErrorConnector
  • know that SQLErrorFactory is used by standard "/handler/..." actions with an implementation called SQLErrorManager which specialize the SQLActionManager.
  • know that SQLErrorFactory is defined using static actions.
    See also
    SQLActionFactory
    SQLActionManager
    SQLErrorManager
    Note
    If you want to implement an ErrorFactoryComponent which uses stateful SQLAction of fixed size try to specialize ObjectActionFactory
