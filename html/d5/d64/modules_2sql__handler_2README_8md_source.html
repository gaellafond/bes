3 # Welcome to the Hyrax sql_handler
4 This handler reads data from a database suing an ODBC driver. There are many
5 different database tools that support access using ODBC.
6 
7 ## Additional Required Software
8 In addition to the Hyrax server, you will need a database that can be accessed using
9 ODBC and an ODBC driver for that database that is compatible with the unixODBC
10 software package. This includes most Relational Database Management Systems (RDBMS)
11 as well as many other interesting tools, including NoSQL databases like MongoDB.
12 
13 If your computer does not have the unixODBC driver software installed, see
14 'Installing ODBC on your Hyrax server' below. To build the Hyrax sql handler, you
15 only need the generic components of unixODBC to be installed. However, to actually
16 serve data from a database, you will need the appropriate (database specific) driver.
17 
18 ## Building and Installing the sql_handler
19 Unlike most of the BES modules, this handler for SQL data is a separate project.
20 It can be added to the bes/modules directory or built outside of the bes software.
21 
22 To build the code:
23 
24 * `autoreconf -fiv`
25 * `./configure --prefix=$prefix`
26 * `make`
27 * `make check` (See more below)
28 * `make install`
29 
30 _Build Notes_
31 * The `./configure` script must find both libdap and the BES. It will use the
32 pkgconfig package manager and, if that fails, look for dap-config and bes-config
33 on the PATH.
34 **configure or make doesn't work**
35 * This will happen if unixODBC is not installed. Only the generic part of unixODBC
36 is needed to configure and build the handler. We suggest you use a binary package
37 from yum, apt-get, or brew (for CentOS, Ubuntu or OSX, respectively).
38 **make check doesn't work**
39 * In order to run `make check` you will need to install the test database located
40 in `install.dir`
41  * Install DBMS as needed (See _Installing DBMS software on your server_ below).
42  * Load database tables in install.dir into DBMS? (See _Creating the test database_ below)
43  * **_how to configure odbc.ini etc._**
44 
45 ## Installing ODBC on your Hyrax server.
46 The `sql_handler` (and by extension Hyrax) uses
47 [Open Database Connectivity (ODBC)](https://en.wikipedia.org/wiki/Open_Database_Connectivity)
48 drivers to access various DBMS systems. Drivers will need to be installed on the Hyrax host system.
49 
50 **[unixODBC](http://www.unixodbc.org)** - This open source ODBC driver is available:
51  - As a source distribution (utilizes gnu autotools to build)
52  - In the `yum` inventory for **CentOS-6** and **CentOS-7** and
53  - In the `homebrew` inventory for **OSX**
54 
55 Some DBMS producers provide their own ODBC drivers. _See the DBMS sections below._
56 
57 ## Configure access to your DBMS
58 In order to serve data from a database you need to have a database running to
59 hold the data and respond to queries. If you haven't got that sorted already,
60 here's some info about how to go about it. In the following, we describe how to
61 configure sqlite for use with ODBC drivers. This is overkill in many ways, but
62 it is an almost universal example. In the appendix we include information about
63 PostGreSQL and MySQL.
64 
65 For this example, we assume you don't have the database installed. Folling these
66 steps you should be able to get a working example.
67 
68 ### Install [SQLite](https://www.sqlite.org) if needed
69 The SQLite binaries are described as:
70 > A bundle of command-line tools for managing SQLite database files, including
71 the command-line shell program, the sqldiff program, and the sqlite3_analyzer
72 program.

88 ## Configure the ODBC drvier and database
89 There are some example `odbc.ini` and `odbcinst.ini` files in the `install.dir`
90 directory, including ones for sqlite's ODBC driver. These two files configure
91 the unixODBC driver.
92 
93 * See [Command Line Shell For SQLite](https://sqlite.org/cli.html) for help with
94 the sqlite command shell.
95 * `odbcinst.ini` - Defines driver options; it creates an association between
96 an odbc driver and a dbms. Location: `/etc/odbcinst.ini` or `/usr/local/etc/odbcinst.ini`
97 * See the [odbcinst](https://www.systutorials.com/docs/linux/man/1-odbcinst/) tool for
98 an alternative way to provide this information (i.e., no editing a config file).
99 * `odbc.ini` - Defines connection options; it creates view into the database
100 and exposes it via the ODBC interface. Location: `/etc/odbc.ini`, `/usr/local/etc/odbc.ini`,
101 or `$HOME/.odbc.ini`
102 
103 The `sqlite_odbcinst.ini` should be renamed to `odbcinst.ini` and copied to `/etc`
104 or `/usr/local/etc`. It holds the following:
105 ```shell script
106 [SQLite]
107 Description=SQLite ODBC Driver
108 Driver=/usr/local/lib/libsqlite3odbc.so
109 Setup=/usr/local/lib/libsqlite3odbc.so
110 Threading=2
111 ```
112 
113 Note that one `odbcinst.ini` can hold information for several database/driver combinations.
114 Separate different sections of the file for different databases using [<name>]
115 
116 The `sqlite_odbc.ini` should be renamed to `odbc.ini` and also copied to /etc or /usr/local/etc.
117 ```shell script
118 [sqlite_test]
119 Description=My SQLite test database
120 Driver=SQLite
121 Database=/Users/jimg/src/opendap/hyrax_git/bes/modules/sql_handler/tests/sqlite_test.db
122 # optional lock timeout in milliseconds
123 Timeout=2000
124 ```
125 
126 * The code includes the `sqlite_test.db` in the `sql_handler/tests/` directory. This
127 database is used for the sqlite-based tests.
128 * There is a SQL script in `install.dir` named `sqlh.sql` that can build the needed
129 database if needed.
130 * The name used in `odbc.ini` (e.g., `[sqlite_test]` above) is used in a 'dataset' file
131 that enables much of SQL to be used when connecting one or more tables from the database to
132 Hyrax via the sql_handler.
133 * Dataset files show up as 'data files' and can be browsed and accessed just
134 like any other OPeNDAP data set.
135 * There are sample 'dataset' files in the sql_hanlder/data directory. Each ends in
136 the extension `.sql` even though they are not SQL program files.
137 * You can change the extensions used by the handler by editing the TypeMatch regular
138 expression in the `sql.conf` file used to configure the handler.
139 
140 * See [isql](https://www.systutorials.com/docs/linux/man/1-isql/) for a command
141 line tool to help debug the ODBC driver configuration.
142 For the test database, `isql sqlite_test` should return the following:
143 ```shell script
144 +---------------------------------------+
145 | Connected! |
146 | |
147 | sql-statement |
148 | help [tablename] |
149 | quit |
150 | |
151 +---------------------------------------+
152 SQL>
153 ```
154 You can enter commands at teh `SQL>` prompt. The command `help sqlh_table` should print
155 information about the table in the test sqlite database.
156 
157 This is one of the sample 'dataset' files (`data/sqlite.sql`):
158 ```shell script
159 # Test variable definitions
160 
161 # Set DSN to the name in square brackets in odbc.init.
162 # For the sqlite example bundled with the handler, that is [sqlite_test].
163 define $DSN$=sqlite_test
164 
165 [section]
166 
167 api=odbc
168 server=$DSN$
169 # The user and pass keywords are not needed
170 # user=root
171 # pass=opendap
172 
173 [select]
174 a, b, c
175 
176 [from]
177 sqlh_table
178 ```
