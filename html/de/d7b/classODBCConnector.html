bool ODBCConnector::close ( void  )
virtual

Close connection.

We end off the transaction using SQLEndTran, and deallocate the handles in reverse order compared with the way we allocated the handles. First we free the statement handle, then we disconnect from the database using SQLDisconnect, and then use SQLFreeHandle to free up all the other handles we allocated!

ODBC_TYPE * ODBCConnector::getNext ( size_t  next = 1)
virtual

Returns a value of ODBC_TYPE type containing the next value in this result set.
It is pointed by the ACTUAL position:

  • column -> getCol()
  • row -> getRow()
    In a Sequence the NEXT value should be:
  • The NEXT COLUMN value of the actual row
    OR
  • The first value of the NEXT ROW
    Note
    This method should also update cursors to the next position using setNext:
  • setNext(size_t increment)
    Note
    The first time it is called should return object in position 0,0 and set cursors to col:1 and row:0 (or row:1 if only 1 column), if limits are reached (col==getCols() && row==getRows()) this method can throw an exception or an SQLInternalException
    use notEnd to check end condition

number of rows to fetch (skip or extract)

Set number of column and rows corresponding to the current position + next. Return the number of skipped rows which is stored for the next fetch operation
