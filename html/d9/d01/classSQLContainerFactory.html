SQLContainer * SQLContainerFactory::getContainer ( BESDataHandlerInterface &  dhi)
static

Build a container starting from the actual selected container of the passed dhi (dhi.container), or get it from the storage (if available). It also take care of the AGE of the stored file calling, (if needed), methods to update the container, substitute the stored copy and return the updated container. This is the method to call if you want to use the ContainerStorage. If you don't, try to use buildContainer.
